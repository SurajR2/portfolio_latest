05  information architecture work / blogs / stack / contact

Site map + section semantics

Four routes. Each is a discrete artifact, not a slide. No "hire me" CTA — contact stands alone. No testimonials, no carousel.

~/
├── /          // landing — name, positioning, latest 3 entries from each section
├── /work      // case studies — system, problem, trade-offs, outcome
│   ├── /:slug    // long-form: context · constraints · architecture · decisions · result
├── /blogs     // technical writing — chronological + tagged
│   ├── /:slug
│   └── /tag/:t   // e.g. /tag/distributed-systems
├── /stack     // runtime, languages, infra, tooling — grouped, not ranked
└── /contact   // email + canonical socials. No form. No "let's chat".

Section content rules

/work
Each entry: context, constraints, architecture diagram, trade-offs considered, outcome (numbers). No screenshots without context. No "I built X" — describe the system and your scope.
/blogs
Long-form. Title states the claim. First paragraph states the conclusion. Code blocks runnable. Tags map to subdomains: systems-design, data, integrations, performance, ops.
/stack
Grouped by layer: runtime · language · datastore · queue · infra · observability · tooling. State competence honestly: production, working, familiar. Skip versions, logos, percentage bars.

07  tone of voice writing rules

Voice

Direct, technical, evidence-led. Senior engineer reviewing an RFC — not a developer-advocate writing a launch post.

Do

  • Lead with the constraint, then the decision.
  • State trade-offs explicitly: "X chosen over Y because Z."
  • Quantify outcomes: latency, throughput, error rate, cost.
  • Use precise nouns: idempotent, backpressure, quorum.
  • Write in plain past tense for completed work.
  • Code blocks must be runnable or marked // pseudocode.

Don't

  • "Passionate developer / lifelong learner / love coding."
  • "Cutting-edge / next-gen / state-of-the-art / robust solutions."
  • Emoji in headings or section dividers.
  • Animated counters for "years of experience" or "projects shipped".
  • Skill bars rated as percentages.
  • "Let's build something amazing together!"
